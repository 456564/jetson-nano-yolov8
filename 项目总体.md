# Jetson Nano A02 YOLOv8n C++ 推理项目

## 项目概况

Jetson Nano (Tegra X1, Maxwell 128核 GPU, SM 5.3) 上部署 YOLOv8n 实时目标检测。
两条推理通道：TensorRT FP16 + CUDA（主打），NCNN FP32 + NEON（备选）。

## 硬件约束

| 特性 | 值 | 影响 |
|---|---|---|
| GPU | 128-core Maxwell | 无 Tensor Core |
| FP16 | 472 GFLOPS | 最优精度 |
| INT8 | 不支持 | SM 5.3 无 DP4A |
| CPU | 4×Cortex-A57 | NEON 128bit SIMD |
| 内存 | 4GB LPDDR4 | 共享 CPU/GPU |

## 8 个核心技术范式

### 1. 全异步 GPU 管道 — 零阻塞
5 个 GPU 操作 (H2D→kernel→enqueue→D2H)，只在最后 sync 一次。CPU 中间完全空闲。

### 2. Pinned Memory
`cudaMallocHost` 锁物理内存，DMA 直接读写，H2D/D2H 延迟从 ~2ms 降到 ~0.5ms。

### 3. GPU 直出指针，零拷贝
`infer()` 返回 `const float*` 指向 pinned memory。每帧省 2.8MB vector::assign。

### 4. CUDA kernel 熔合 5 道工序
resize + pad + BGR→RGB + normalize + HWC→CHW 一个 kernel 完成。省 3 次显存读写。

### 5. Cache-unfriendly 换并行
后处理从"类优先串行"改为"box 优先 4 核并行"。3× 并行收益 > cache miss 代价。

### 6. 推理/显示线程分离 (NCNN 版)
后台线程推理，主线程只画框 + X11 事件循环。mutex 持有时间 < 0.1ms。

### 7. 预热 + 分阶段性能剖析
前 5 帧丢弃（冷 cuDNN kernel cache），每 30 帧打印 4 段耗时。不分段不配调优。

### 8. NCNN NEON 双开关
`num_threads=4` + `use_packing_layout=true`。一行代码，ARM 推理速度翻倍。

## 构建与运行

```bash
# 1. 克隆（含 ncnn submodule）
git clone <repo-url> jetson-nano-yolov8
cd jetson-nano-yolov8
git submodule update --init

# 2. 构建
cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j4

# 3. 运行 TensorRT 主程序
./detect

# 4. 或运行 NCNN 独立版本
cd ..
python scripts/export_onnx.py     # 先导出 ONNX
# 然后用 tools/pnnx 转换 → 或用预置的 ncnn 模型
```
